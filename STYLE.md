# Style Guide

이 위키의 모든 본문 페이지가 따르는 단일 진실 소스입니다. LLM 은 본문 작성 전에 항상 이 문서를 확인합니다.

## 문체

- **격식체** (`~합니다` / `~입니다`)
- **2인칭**: `여러분` 으로 통일 (원문 "you" 처리)
- **명령형** (`~하세요`) 은 서술 (`~합니다` / `~해야 합니다`) 로 변환

## 전문용어

- 첫 등장 시 `한국어(원어)` 형태로 병기. 이후 같은 페이지 안에서는 한국어만 사용.
- 한국어 정착어가 없으면 원어 유지 (예: hook).
- 약어는 그대로 유지 (API, REST, HTTP, URL, JSON 등).

## 코드 블록

- 코드 자체는 원문 그대로.
- 인라인 주석 (`// ...`, `# ...`) 은 번역.
- 다중 줄 설명 주석은 원어 유지 + 아래에 번역 병기.

## 본문 충실성

- 원문의 헤딩 구조, 단락 순서, 코드 예시 위치, 이미지 위치, 강조(굵게/이탤릭) 위치를 그대로 유지합니다.
- 의역·번역자 주석·재구성은 본문에 넣지 않습니다. 필요한 정리는 본문 바깥(채팅, 향후 `_notes/`)에서만.

## 용어 결정 표

| 원어 | 한국어 정착어 | status | 결정일 | 비고 |
|---|---|---|---|---|

상태값:

- `정착어`: 한국어로 옮긴다
- `원어유지`: 원어 그대로 쓴다
- `미정`: 아직 결정하지 않음 (본문에서는 원어 유지하며 결정 대기)

새 용어가 본문에 등장하면 LLM 이 본문 작성 전에 위 표에 항목을 추가하고, 동시에 `_glossary/<term>.md` 를 생성·갱신합니다.
